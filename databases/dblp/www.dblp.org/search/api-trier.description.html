DBLP Search API Trier

This is a new, experimental API, currently used and tested by the DBLP team in Trier. In the following it will be explained by example. Just click on the link and look at the result, this will make most things clear enough.

Basic one-word query: ?q=algorithm (query word is matched anywhere in the record)
Basic multi-word query: ?q=graph+algorithm (query words are matched anywhere in the record)
Specify range of hits : ?q=graph*+algo*&f=0&h=2 (the f specifies the index of the first hit, h specifies the number of hits)
Prefix query and completions: ?q=graph*+algo* (will show completions for last query word that lead to hits)
Specify number of completions: ?q=graph*+algo*&c=30
... can be arbitrarily combined: ?q=graph*+algo*&h=0&c=30
Facets for a particular query: ?q=graph*+algo*+:facet:author:* (facets available are: author, venue, year, type)
Search only in a particular field: ?q=:filter:title:mehlhorn (filters available are: title, author, year, type, and see below for authorname)
... can be arbitrarly combined: ?q=:filter:title:mehlh*
+:filter:author:rytter+algo*
Error-tolerant (fuzzy) search: ?q=algrithm~ (the ~ can be added or not separately for each query word)
... also works for fielded search: ?q=:filter:author:melhorn~
Disable normalization: ?q="müller" (the quotes can surround letters or whole words and can be used multiple times)
... can also be used to disable the interpretation of special characters: ?q=:facet:author:H"."_Vincent_Poor
Search for author names only: ?q=:filter:authorname:stefan~
+:filter:authorname:funke~
+:facet:authorname:*&h=0&c=100		 (understand that :filter:author:... searches all authors for each record; so if you are looking for matches within one and the same author name, you must use ...:authorname:...)
